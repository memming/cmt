Class MLR

An implementation of multinomial logistic regression (also known as softmax regression).

$$p(\mathbf{y} \mid \mathbf{x}) = \frac{\exp(\mathbf{w}_i^\top \mathbf{x} + b_i)}{\sum_j \exp(\mathbf{w}_j^\top \mathbf{x} + b_j)},$$

for $\mathbf{y} \in \{0, 1\}^N$ with $y_i = 1$ and $y_j = 0$ for all $j \neq i$.

To access linear filters $\mathbf{w}_i$ and biases $b$, use

>>> mlr = MLR(M, N)
>>> mlr.weights
>>> mlr.biases

train(self, input, output, input_val=None, output_val=None, parameters=None)

source code 

Fits model parameters to given data using L-BFGS. Because the parameters are redundant, the filter and bias belonging to the first output are fixed and only the remaining parameters are optimized.

The following example demonstrates possible parameters and default settings.

>>> model.train(input, output, parameters={
>>>     'verbosity': 0,
>>>     'max_iter': 1000,
>>>     'threshold': 1e-9,
>>>     'num_grad': 20,
>>>     'batch_size': 2000,
>>>     'callback': None,
>>>     'cb_iter': 25,
>>>     'val_iter': 5,
>>>     'val_look_ahead': 20,
>>>     'train_weights': True,
>>>     'train_biases': True,
>>>     'regularize_weights': {
>>>             'strength': 0.,
>>>             'transform': None,
>>>             'norm': 'L2'},
>>>     'regularize_biases': {
>>>             'strength': 0.,
>>>             'norm': 'L2'},
>>> })

The optimization stops after max_iter iterations or if the difference in (penalized) log-likelihood is sufficiently small enough, as specified by threshold. num_grad is the number of gradients used by L-BFGS to approximate the inverse Hessian matrix.

Regularization of parameters $\mathbf{z}$ adds a penalty term

$$\eta ||\mathbf{A} \mathbf{z}||_p$$

to the average log-likelihood, where $\eta$ is given by strength, $\mathbf{A}$ is given by transform, and $p$ is controlled by norm, which has to be either 'L1' or 'L2'.

The parameter batch_size has no effect on the solution of the optimization but can affect speed by reducing the number of cache misses.

If a callback function is given, it will be called every cb_iter iterations. The first argument to callback will be the current iteration, the second argument will be a copy of the model.

>>> def callback(i, mlr):
>>>     print i
Parameters:
  • input (ndarray) - inputs stored in columns
  • output (ndarray) - outputs stored in columns
  • input_val (ndarray) - inputs used for early stopping based on validation error
  • output_val (ndarray) - outputs used for early stopping based on validation error
  • parameters (dict) - a dictionary containing hyperparameters
Returns: bool
True if training converged, otherwise False
