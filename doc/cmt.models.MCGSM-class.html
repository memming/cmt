Class MCGSM

source code

An implementation of a mixture of conditional Gaussian scale mixtures.

The distribution defined by the model is

$$p(\mathbf{y} \mid \mathbf{x}) = \sum_{c, s} p(c, s \mid \mathbf{x}) p(\mathbf{y} \mid c, s, \mathbf{x}),$$

where

\begin{align} p(c, s \mid \mathbf{x}) &\propto \exp\left(\eta_{cs} - \frac{1}{2} e^{\alpha_{cs}} \sum_i \beta_{ci}^2 \left(\mathbf{b}_i^\top \mathbf{x}\right)^2 + e^{\alpha_{cs}} \mathbf{w}_c^\top \mathbf{x} \right),\\ p(\mathbf{y} \mid c, s, \mathbf{x}) &= |\mathbf{L}_c| \exp\left(\frac{M}{2}\alpha_{cs} - \frac{1}{2} e^{\alpha_{cs}} (\mathbf{y} - \mathbf{A}_c \mathbf{x} - \mathbf{u}_c)^\top \mathbf{L}_c \mathbf{L}_c^\top (\mathbf{y} - \mathbf{A}_c \mathbf{x} - \mathbf{u}_c)\right) / (2\pi)^\frac{M}{2}. \end{align}

To create an MCGSM for $N$-dimensional inputs $\mathbf{x} \in \mathbb{R}^N$ and $M$-dimensional outputs $\mathbf{y} \in \mathbb{R}^M$ with, for example, 8 predictors $\mathbf{A}_c$, 6 scales $\alpha_{cs}$ per component $c$, and 100 features $\mathbf{b}_i$, use

>>> mcgsm = MCGSM(N, M, 8, 6, 100)

To access the different parameters, you can use

>>> mcgsm.priors
>>> mcgsm.scales
>>> mcgsm.weights
>>> mcgsm.features
>>> mcgsm.cholesky_factors
>>> mcgsm.predictors
>>> mcgsm.linear_features
>>> mcgsm.means

which correspond to $\eta_{cs}$, $\alpha_{cs}$, $\beta_{ci}$, $\mathbf{b}_i$, $\mathbf{L}_c$, and $\mathbf{A}_c$, respectively.

References:

  • L. Theis, R. Hosseini, M. Bethge, Mixtures of Conditional Gaussian Scale Mixtures Applied to Multiscale Image Representations, PLoS ONE, 2012.

train(self, input, output, input_val=None, output_val=None, parameters=None)

source code 

Fits model parameters to given data using L-BFGS.

The following example demonstrates possible parameters and default settings.

>>> model.train(input, output, parameters={
>>>     'verbosity': 0,
>>>     'max_iter': 1000,
>>>     'threshold': 1e-9,
>>>     'num_grad': 20,
>>>     'batch_size': 2000,
>>>     'callback': None,
>>>     'cb_iter': 25,
>>>     'val_iter': 5,
>>>     'val_look_ahead': 20,
>>>     'train_priors': True,
>>>     'train_scales': True,
>>>     'train_weights': True,
>>>     'train_features': True,
>>>     'train_cholesky_factors': True,
>>>     'train_predictors': True,
>>>     'train_linear_features': False,
>>>     'train_means': False,
>>>     'regularize_features': {
>>>             'strength': 0.,
>>>             'transform': None,
>>>             'norm': 'L2'},
>>>     'regularize_weights': {
>>>             'strength': 0.,
>>>             'transform': None,
>>>             'norm': 'L2'},
>>>     'regularize_predictors': {
>>>             'strength': 0.,
>>>             'transform': None,
>>>             'norm': 'L2'},
>>>     'regularize_linear_features': {
>>>             'strength': 0.,
>>>             'transform': None,
>>>             'norm': 'L2'},
>>>     'regularize_means': {
>>>             'strength': 0.,
>>>             'transform': None,
>>>             'norm': 'L2'},
>>> })

The parameters train_priors, train_scales, and so on can be used to control which parameters will be optimized. Optimization stops after max_iter iterations or if the gradient is sufficiently small enough, as specified by threshold.num_grad is the number of gradients used by L-BFGS to approximate the inverse Hessian matrix.

Regularization of parameters $\mathbf{z}$ adds a penalty term

$$\eta ||\mathbf{A} \mathbf{z}||_p$$

to the average log-likelihood, where $\eta$ is given by strength, $\mathbf{A}$ is given by transform, and $p$ is controlled by norm, which has to be either 'L1' or 'L2'.

The parameter batch_size has no effect on the solution of the optimization but can affect speed by reducing the number of cache misses.

If a callback function is given, it will be called every cb_iter iterations. The first argument to callback will be the current iteration, the second argument will be a copy of the model.

>>> def callback(i, mcgsm):
>>>     print i
Parameters:
  • input (ndarray) - inputs stored in columns
  • output (ndarray) - outputs stored in columns
  • input_val (ndarray) - inputs used for early stopping based on validation error
  • output_val (ndarray) - outputs used for early stopping based on validation error
  • parameters (dict) - a dictionary containing hyperparameters
Returns: bool
True if training converged, otherwise False
